Skills-System-Anleitung

Wiederverwendbare Claude-Prompts zur Automatisierung komplexer Workflows

🧩 Was sind Skills?

Skills sind wiederverwendbare Claude-Prompts, die komplexe, mehrstufige Workflows automatisieren. Jeder Skill ist eine strukturierte SKILL.md-Datei, gespeichert in ~/.claude/skills/, und wird direkt aus Claude Code aufgerufen.

📋

Strukturierte Workflows

Mehrstufige Prozesse mit Gates und Checkpoints

🔄

Wiederverwendbar

Einmal installieren, mit Variablenersetzung projektübergreifend nutzen

🤖

KI-gesteuert

Claude folgt den Skill-Anweisungen automatisch über MCP

🚀 Schnellstart 3 Schritte

Skills in wenigen Minuten einrichten und verwenden.

1

Verfügbare Skills anzeigen

list_skills()

Zeigt alle verfügbaren und installierten Skills mit ihrem Status an.

2

Mit Variablen installieren

install_skill({
  name: "story-engine",
  variables: { PROJECT: "/path/to/episode" }
})

Variablen wie {{PROJECT}} werden bei der Installation durch tatsächliche Werte ersetzt. Enthält die Genres yadam (Koreanisch) und dark-history (Englisch).

3

In Claude Code verwenden

Der installierte Skill erscheint in der Claude Code Befehlspalette. Claude folgt automatisch dem im Skill definierten Workflow.

📖 Story Engine v0.9

Automatisierte Drehbucherstellung für YouTube-Story-Kanäle in einem 12-Schritte-Workflow. Entwickelt für traditionelle Erzählungen, Volksmärchen und historische Geschichten. Deckt die gesamte Pipeline von der Story-Analyse bis zu upload-fertigen Assets ab.

12-Schritte-Workflow Drehbuch TTS / SFX Storyboard CSV Bild- / Videogenerierung

Variablen

PROJECT (erforderlich) — Pfad zum Episodenordner

🎭 Story Engine Genres

Das story-engine Skill unterstützt zwei Genres, die automatisch anhand Ihrer Sprache ausgewählt werden:

Genre Sprache Stil Beispiele
yadam Koreanisch Koreanische historische Erzählungen — Joseon-Dynastie, Volksmärchen "야담 대본 써줘"
dark-history English Westliche dunkle Geschichte — Mittelalter-Mysterien, Tudor-Intrigen, Gothic "Write a script about Salem witch trials"

Automatische Erkennung: Das Genre wird automatisch anhand Ihrer Eingabesprache erkannt. Verwenden Sie --genre yadam oder --genre dark-history zur manuellen Auswahl.

yadam Meta-Prompts

5 koreanische Meta-Prompt-Dateien: Synopsis, Preflight, Drehbuch, Narration, Spannung

dark-history Meta-Prompts

5 englische Meta-Prompt-Dateien basierend auf Google DeepMinds Dramatron-Framework

🔄 12-Schritte-Workflow

Die Story Engine folgt einer strikten 12-Schritte-Pipeline mit Gate-Prüfungen zwischen den Phasen.

R1

Story-Analyse

Analyse einer Referenzgeschichte auf Erfolgsfaktoren, Zuschauer-Hooks und Erzählstruktur.

R2

Faktenprüfung

Überprüfung historischer Genauigkeit, zeitgemäßer Details und inhaltlicher Konsistenz.

R3

Synopsis

Erstellung eines 20-Kapitel-Frameworks mit Plotstruktur und Charakterbögen.

R4

Preflight

Strukturprüfung, Verbotsinhalte-Check und Foreshadowing-Tracking.

R5

Drehbuch schreiben

Vollständiges Drehbuch in 5 Teilen: Einleitung → Entwicklung → Wendung → Auflösung → Hook.

R6

Überprüfung

Iterative Sub-Agent-Überprüfungsrunden (bis zu 5 Runden) zur Qualitätssicherung.

R7

Finalisierung Benutzerfreigabe erforderlich

Benutzerfreigabe-Checkpoint. Ohne explizite Bestätigung kann nicht zur Produktion übergegangen werden.

R8

Produktionsextraktion

Aufteilung des Drehbuchs in Erzählung, Charakterstimmen und SFX-Cues.

R8.5

Extraktionsprüfung

Sub-Agent-Validierung der extrahierten Produktionsdaten.

R9

TTS / SFX-Generierung

Audiodateien mit KI-TTS-APIs (ElevenLabs, Typecast) und Soundeffekte generieren.

R10

Storyboard CSV

Erstellung von references.csv und scenes.csv für die Bild-/Videogenerierung mit AutoFlowCut.

R11

Bild- / Videogenerierung

Batch-Generierung mit Flow AI über AutoFlowCut MCP-Tools und Qualitätssicherung.

R12

Upload-Informationen

Vorbereitung von Titel, Beschreibung, Tags und Thumbnail für den YouTube-Upload.

⚙️ Skill-Verwaltung

list_skills()

Alle verfügbaren und installierten Skills mit Status und Metadaten anzeigen.

install_skill(name, variables)

Skill mit Template-Variablenersetzung installieren. Die Skill-Datei wird nach ~/.claude/skills/ kopiert.

uninstall_skill(name)

Installierten Skill aus dem Skills-Verzeichnis entfernen.

🛠️ Eigene Skills erstellen

Sie können eigene Skills erstellen, um beliebige Workflows zu automatisieren.

Dateistruktur

skills/
  {skill-name}/
    SKILL.md          # Workflow-Anweisungen
    metadata.json     # Name, Beschreibung, Variablen
    docs/             # Referenzdokumente (optional)

Template-Variablen

Verwenden Sie {{VAR_NAME}}-Platzhalter in SKILL.md. Sie werden bei der Installation durch tatsächliche Werte ersetzt.

Beispiel: {{PROJECT}} in der Skill-Datei wird nach der Installation zu /Users/me/workspace/ep01.
