🎯 스킬

스킬 시스템 가이드

복잡한 워크플로우를 자동화하는 재사용 가능한 Claude 프롬프트

🧩 스킬이란?

스킬은 복잡한 다단계 워크플로우를 자동화하는 재사용 가능한 Claude 프롬프트입니다. 각 스킬은 ~/.claude/skills/에 저장된 SKILL.md 파일이며, Claude Code에서 직접 호출할 수 있습니다.

📋

체계적 워크플로우

게이트와 체크포인트가 있는 다단계 프로세스

🔄

재사용 가능

한 번 설치하고 변수 치환으로 여러 프로젝트에서 사용

🤖

AI 기반

Claude가 MCP를 통해 스킬 지침을 자동으로 따릅니다

🚀 빠른 시작 3단계

몇 분 만에 스킬을 설정하고 사용할 수 있습니다.

1

사용 가능한 스킬 확인

list_skills()

사용 가능한 스킬과 설치된 스킬의 상태를 확인합니다.

2

변수와 함께 설치

install_skill({
  name: "story-engine",
  variables: { PROJECT: "/path/to/episode" }
})

{{PROJECT}} 같은 변수가 설치 시 실제 경로로 치환됩니다. yadam(한국어)과 dark-history(영어) 두 장르를 포함합니다.

3

Claude Code에서 사용

설치된 스킬은 Claude Code 명령 팔레트에 나타나며, Claude가 스킬에 정의된 워크플로우를 자동으로 따릅니다.

📖 Story Engine v0.9

유튜브 스토리 채널용 대본을 12단계 워크플로우로 작성합니다. 야담, 민담, 설화 등 전통 이야기 대본부터 프로덕션 에셋까지 전체 파이프라인을 커버합니다.

12단계 워크플로우 대본 작성 TTS / SFX 스토리보드 CSV 이미지 / 영상 생성

변수

PROJECT (필수) — 에피소드 폴더 경로

🎭 Story Engine 장르

story-engine 스킬은 2개의 장르를 지원하며, 입력 언어에 따라 자동 선택됩니다:

장르 언어 스타일 예시
yadam 한국어 조선시대 야담, 민담, 설화, 궁중 미스터리, 기담 "야담 대본 써줘", "ep11 시작하자"
dark-history English 서양 다크 히스토리 — 중세 미스터리, 튜더 왕조, 고딕 테일, 실화 범죄 "Write a script about Salem witch trials"

자동 감지: 장르는 입력 언어에서 자동 감지됩니다. --genre yadam 또는 --genre dark-history로 수동 선택할 수 있습니다.

yadam 메타 프롬프트

5개의 한국어 메타 프롬프트 파일: 시놉시스, 사전점검, 극본, 내레이션, 서스펜스

dark-history 메타 프롬프트

Google DeepMind Dramatron 프레임워크 기반 5개의 영어 메타 프롬프트 파일

🔄 12단계 워크플로우

Story Engine은 단계 간 게이트 체크가 있는 엄격한 12단계 파이프라인을 따릅니다.

R1

성공요인 분석

레퍼런스 스토리의 성공 요인, 시청자 후킹 포인트, 서사 구조를 분석합니다.

R2

팩트체크

시대고증, 역사적 사실, 내용의 일관성을 검증합니다.

R3

시놉시스

플롯 구조와 캐릭터 아크를 포함한 20챕터 프레임워크를 작성합니다.

R4

프리플라이트

구조 점검, 금지사항 확인, 복선 추적표를 작성합니다.

R5

대본 작성

기 → 승 → 전 → 결 → 훅 5파트로 전체 대본을 작성합니다.

R6

검토

서브에이전트를 통한 반복 검토 (최대 5라운드)로 품질을 보장합니다.

R7

대본 확정 사용자 승인 필수

사용자 승인 체크포인트입니다. 명시적 확인 없이는 프로덕션으로 진행할 수 없습니다.

R8

프로덕션 추출

대본을 나레이션, 인물 대사, SFX 큐로 분리합니다.

R8.5

추출 검토

서브에이전트가 추출된 프로덕션 데이터를 검증합니다.

R9

TTS / SFX 생성

AI TTS API (ElevenLabs, Typecast)로 음성 파일과 효과음을 생성합니다.

R10

스토리보드 CSV

AutoFlowCut으로 이미지/영상 생성을 위한 references.csv와 scenes.csv를 작성합니다.

R11

이미지 / 영상 생성

AutoFlowCut MCP 도구로 Flow AI 배치 생성 및 품질 검수를 진행합니다.

R12

업로드 정보

유튜브 업로드를 위한 제목, 설명, 태그, 썸네일을 준비합니다.

⚙️ 스킬 관리

list_skills()

사용 가능한 모든 스킬과 설치된 스킬의 상태 및 메타데이터를 확인합니다.

install_skill(name, variables)

템플릿 변수 치환과 함께 스킬을 설치합니다. 스킬 파일이 ~/.claude/skills/에 복사됩니다.

uninstall_skill(name)

설치된 스킬을 스킬 디렉토리에서 제거합니다.

🛠️ 커스텀 스킬 만들기

어떤 워크플로우든 자동화하는 나만의 스킬을 만들 수 있습니다.

파일 구조

skills/
  {skill-name}/
    SKILL.md          # 워크플로우 지침
    metadata.json     # 이름, 설명, 변수 정의
    docs/             # 참조 문서 (선택사항)

템플릿 변수

SKILL.md에서 {{VAR_NAME}} 플레이스홀더를 사용합니다. 설치 시 실제 값으로 치환됩니다.

예시: 스킬 파일의 {{PROJECT}}가 설치 후 /Users/me/workspace/ep01로 변환됩니다.
