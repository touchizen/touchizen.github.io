🧩 스킬이란?

스킬은 복잡한 다단계 워크플로우를 자동화하는 재사용 가능한 Claude 프롬프트입니다. 각 스킬은 ~/.claude/skills/에 저장된 SKILL.md 파일이며, Claude Code에서 직접 호출할 수 있습니다.

PROJECT (필수) — 에피소드 폴더 경로

🎭 Story Engine 장르

story-engine 스킬은 3개의 장르를 지원하며, 입력 언어와 주제 키워드에 따라 자동 선택됩니다:

장르 언어 스타일 예시
야담(yadam) 한국어 조선시대 야담, 민담, 설화, 궁중 미스터리, 기담 "야담 대본 써줘", "ep11 시작하자"
다크 히스토리(dark-history) English 서양 다크 히스토리 — 중세 미스터리, 튜더 왕조, 고딕 테일, 실화 범죄 "Write a script about Salem witch trials"
맞춤형(bespoke) 모든 언어 맞춤형 — 사용자가 제공한 성공 대본 3~5개로 메타 프롬프트를 합성. 어떤 주제·언어든 가능 "경제 유튜브 대본 써줘" (참고 대본 3~5개 필요)

자동 감지: 장르는 입력 언어와 주제 키워드에서 자동 감지됩니다. --genre yadam, --genre dark-history, --genre bespoke로 수동 선택할 수 있습니다.

야담(yadam) 메타 프롬프트

5개의 한국어 메타 프롬프트 파일: 시놉시스, 사전점검, 극본, 내레이션, 서스펜스

다크 히스토리(dark-history) 메타 프롬프트

Google DeepMind Dramatron 프레임워크 기반 5개의 영어 메타 프롬프트 파일

맞춤형(bespoke) 메타 프롬프트

출력 언어별 ko/en 각 5개 파일 + 에피소드마다 사용자 레퍼런스 3~5개에서 합성한 보충 프롬프트

🔄 9-Wave 워크플로우

Story Engine은 단계 간 게이트 체크가 있는 엄격한 9-Wave 파이프라인을 따릅니다.

W1

스토리 설계

레퍼런스 분석과 팩트체크로 패턴·강점을 파악하고 스토리 방향을 잡습니다. 신규 제작과 리라이팅을 분기합니다.

W2

시놉시스 + 프리플라이트

20챕터 기승전결훅 시놉시스를 작성하고, 집필 전 구조·복선·서스펜스를 점검합니다.

W3

대본 작성 + 검토 사용자 승인 필수

기 → 승 → 전 → 결 → 훅 5파트로 대본을 작성하고 서브에이전트가 최대 5라운드 검토합니다. 완료 후 사용자 승인 체크포인트입니다.

W4

프로덕션 추출

대본을 나레이션, 인물 대사, SFX 큐로 분리하고 검증합니다.

W5

TTS / SFX 생성

AI TTS API (ElevenLabs, Typecast)로 음성과 효과음을 생성하고 타임코드를 검증합니다.

W6

스토리보드 CSV

AutoFlowCut 이미지/영상 생성을 위한 references.csv와 scenes.csv를 작성하고 배치 QA로 검토합니다.

W7

이미지 프로덕션 사용자 승인 필수

AutoFlowCut MCP로 레퍼런스·씬 이미지/영상을 배치 생성하고 이미지 QA를 진행합니다. 완료 후 사용자 승인 체크포인트입니다.

W8

어셈블리

SFX 씬 매칭, 오디오 임포트, CapCut 내보내기로 편집 프로젝트를 조립합니다.

W9

업로드 정보

유튜브 업로드를 위한 제목, 설명, 태그, 썸네일을 준비합니다.

⚙️ 스킬 관리

list_skills()

사용 가능한 모든 스킬과 설치된 스킬의 상태 및 메타데이터를 확인합니다.

install_skill(name, variables)

템플릿 변수 치환과 함께 스킬을 설치합니다. 스킬 파일이 ~/.claude/skills/에 복사됩니다.

uninstall_skill(name)

설치된 스킬을 스킬 디렉토리에서 제거합니다.

🛠️ 커스텀 스킬 만들기

어떤 워크플로우든 자동화하는 나만의 스킬을 만들 수 있습니다.

파일 구조

skills/
  {skill-name}/
    SKILL.md          # 워크플로우 지침
    metadata.json     # 이름, 설명, 변수 정의
    docs/             # 참조 문서 (선택사항)

템플릿 변수

SKILL.md에서 {{VAR_NAME}} 플레이스홀더를 사용합니다. 설치 시 실제 값으로 치환됩니다.

예시: 스킬 파일의 {{PROJECT}}가 설치 후 /Users/me/workspace/ep01로 변환됩니다.
